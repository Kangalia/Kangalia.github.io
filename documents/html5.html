CHAPTER 1: HISTORY OF HTML

  • HTML was the brainchild of Sir Tim, getting its origins from the Standard Generalized Markup Language (SGML).
  • The first official specification was published by the Internet Engineering Task Force was HTML 2.0. The role of the IETF was superceded by the W3Consortium, which published the HTML 4.01.
  • The next revision was XHTML 1.0 (extensible HTML) (The XHTML only allowed for tags to be written in small letters. There was browser support for CSS).
  • Then there was XHTML 1.1 which couldn’t be served with a mime-type of text/html. Later, there was XHTML 2.0 which was not backward compatible with current content then
  • As an aggressive rebellion against this principle of strictness, the Web Hypertext Application Technology Working Group (WHATWG) championed the creation of what is now HTML5.
  • The WHATWG is creating an HTML5 specification using its process of "commit then review", while the W3C HTML Working Group is taking that specification and putting it through its process of "review then commit".

CHAPTER 2: DESIGN PRINCIPLES OF HTML5

THE DESIGN PRINCIPLES

  • HTML5 provides support for previous content
  • Do not reinvent the wheel
  • Pave the cowpaths. "If it is not broken, do not fix it"

SIMPLICITY

  • Unlike in other previous versions of HTML and XHTML declaring the document type is easy: 
    !DOCTYPE html
    This allows for backward compatibility with browsers that support previous versions of HTML and XHTML
  • In defining the character encoding in HTML5, you do it as so: 
    meta charset="UTF-8"
    in order to ensure the server sends the correct Content Type header.
  • In defining the "script" you do so as without adding the 
    type="text/javascript"
    statement in the opening "script" tag
  • Same goes for the stylesheet where you don't need to put the 
    type="text/css"
    statement in the opening "link" tag

OBSOLETE AND DEPRECATED ELEMENTS AND ATTRIBUTES

  • DEPRECATED means no longer used, while OBSOLETE means it's not used in modern browsers but is used anyway for the sake of older browsers
  • Some of the obsolete elements include: frame, frameset, noframes and acronym.
  • Presentational elements such as font, big, center and strike are also obsolete
  • Similarly, presentational attributes such as bgcolor , cellspacing, cellpadding and valign are obsolete.
  • The big element is obsolete but the small element isn’t. This apparent inconsistency has been resolved by redefining what "small" means. It no longer has the presentational connotation, "render this at a small size." Instead, it has the semantic value, "this is the small print," for legalese, or terms and conditions.
  • The b element used to mean, "render this in bold". Instead it is used for some text "to be stylistically offset from the normal prose without conveying any extra importance". If the text has any extra importance, then the strong element would be more appropriate. Similarly, the i element no longer means "italicize." For emphasis, use the em element.
  • The cite element has been redefined in HTML5. Where it previously meant "a reference to other sources", it now means "the title of a work".
  • In HTML5, you can wrap multiple elements in a single a element.

CHAPTER 5: SEMANTICS

EXTENSIBILITY, NEW ELEMENTS, STRUCTURE AND CONTENT MODELS

  • Elements in HTML5 can be extended using the "class" attribute This is similar to RDFa's microformats (microformats are conventions which have been unanimously agreed upon by the community) 
                Extensibility semantic in HTML5
            h1 class="summary"
        
                Extensibility semantic in RDFa
            h1 property="myformat:summary"
  • Some new text-level semantics have also been added to vast collection of span , strong , em , abbr , etc
    1. Mark - for highlighting
    My name is Isaac Kang'alia
    2. Time - used for both dates and times
    January 12th, 1992
    3. Meter - can be used to mark up measurements, provided that those measurements are part of a scale with minimum and maximum values.
    9 out of 10 cats 9 out of 10 cats
    4. Progress - allows you to mark up a value that is in the process of changing. You can add min max and value attributes to the opening progress element if you like
    Your profile is 60% complete. Your profile is 60% complete.
  • The section element is used for grouping together thematically-related content. While the div has no semantic meaning, i.e. it is simply a generic container, a section is used explicitly for grouping related content.
  • The header element is a container for content in the masthead. This means that there is only one masthead but there can be more than one header in it. You can also use the header element within a section element, for example.
  • The footer element contains information about its containing element: who wrote it, copyright information, links to related content, etc. Just like header elements HTML5 allows for more than one footer within a section
  • The aside matches the concept of a sidebar. But it’s the content that matters, not the position. The aside element should be used for tangentially related content. If you have a chunk of content that you consider to be separate from the main content, then the aside element is probably the right container for it. Pullquotes are a good example of tangentially related content; they’re nice to have, but you can remove them without affecting the comprehension of the main content.
  • The nav element contains navigation information, usually a list of links. The nav element is intended for major navigation information such as site-wide navigation. Quite often, a nav element will appear within a header element.
  • The article element is another specialized kind of section and is used for self-contained related content. The article element is specifically designed for syndication. If you use a time element within an article , you can add an optional pubdate Boolean attribute to indicate that it contains the date of publication. The article element is useful for blog posts, news stories, comments, reviews, and forum posts. It covers exactly the same use cases as the hAtom microformat. The article element should be used for self contained widgets: stock tickers, calculators, clocks, weather widgets, and the like.
  • The hgroup element allows you to use heading elements without having them appear in the document outline. The blockquote , fieldset , and td elements are all immune to the outline algorithm. These elements are called “sectioning roots”
